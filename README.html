A. Contents of this package

Directory/File Contents
. Top-level directory files include:
  • README.html (this document)
./feature Contains data files used to build and/or run the system:
  • ant/ contains the optional taskdefs needed for supplemental tasks. 
  • checkstyle/  contains the checkstyle.xml file
  • csdl/ contains the Welcome.html page for the Jupiter web site.
  • jar/ contains the jar files need to compile the system
  • log4j/ contains the property file for the log4j, which is used in Jupiter plug-in
  • update/ contains the files for the update manger strucuture.
./plugin Contains developer documentation including:
  • History.html provides a chronological record of the releases of this system
  • InstallationGuide.html provides the installation guide for the zip-based Jupiter plug-in package
  • UsersGuide.html provides the user's guide for the Jupiter plug-in
./site Contains the source files.
./build Contains various products of the build process, including:
  • javadoc/: Contains javadoc API information, created by 'ant javadoc'.
  • locc/: Contains size data on the system, created by 'ant locc'
  • junit/: Contains output from the JUnit tests, created by 'ant test'
  • java2html:/ Contains HTML version of the source code, created by 'ant java2html'
  • dist/ Contains any distributions built during development.

These links won't work until their corresponding ant target has been executed.

B. Installation

These instructions assume that you have unzipped the zip file containing this system to create a directory containing the system components. A note to Unix users: due to a bug in java.util.zip, the directory permissions are munged if you unzip in Unix, and you won't be able to cd into interior directories.  To fix this, invoke the following from the parent directory of this system: chmod -R a+x <system directory>. 

  1. Download and install the Java SDK version 1.4 or above.  
    • The canonical site for obtaining Java SDKs is http://java.sun.com/.
        
  2. Set the JAVA_HOME environment variable to the top-level directory of the Java SDK. 
    • The procedure for setting environment variables is different on Windows, Mac, and Unix; consult your sysadmin or google if you do not know how to set environment variables. 
    • An example value for JAVA_HOME on a windows environment is: c:\jdk1.4.2.
        
  3. Download and install Ant 1.5.4 or above.
    • The canonical site for obtaining Ant is http://ant.apache.org/. 
       
  4. Set the ANT_HOME environment variable and add the Ant scripts to your path.
    • Define the ANT_HOME environment variable An example value is c:\apache-ant-1.5.4. 
    • Add the <ANT_HOME>/bin directory to your PATH variable.  For details, see the Ant installation instructions.
        
  5. Add additional libraries to your Ant installation by invoking 'ant updateAntLib'. 
    • This copies files from the lib/ant directory (and junit.jar) to ANT_HOME/lib and thus makes additional tasks available.
        
  6. Download and install Tomcat 4.1.x Light Edition (not Tomcat 5.x).
    • Define the CATALINA_HOME environment variable. An example value is c:\jakarta-tomcat-4.1.24-LE-jdk14
    • Use the light edition (LE) because we are using JDK 1.4 and above.
    • It is usually convenient to add <tomcat>/bin to your PATH variable.
        
  7. Edit the <tomcat>/conf/tomcat-users.xml file to include an "jupiter" user with a password.
    • This is needed to provide Ant-based access to the Tomcat Manager application.
    • An example tomcat-users.xml file is provided in sample.tomcat-users.xml.
    • If you change the password or account name, update the build.xml properties appropriately.
        
  8. Bring up Tomcat using its startup script by typing "startup".
      
  9. Invoke 'ant checkTomcat'.
    • This checks that you have Tomcat running and that Ant can access the Tomcat Manager application.

C. Invocation

Using the Jupiter system involves building it, deploying it to the Tomcat webserver webapps directory, starting up Tomcat, and then viewing the Jupiter web site page in a browser:
  • Change directory to the jupiter home directory. For e.g., c:\cvs\jupiter
  • Invoke 'startup' to start up Tomcat.
  • Invoke 'ant' to install the jupiter web site.
  • Visit the Welcome.html page. For a default Tomcat installation on the local host, this is located at: http://localhost:8080/Jupiter/Welcome.html.
  • There are many functions that the Jupiter system provides. Please see the user documentation below.

E. Developer documentation

  • The Jupiter build system is based on Jakarta Ant. The build.xml file contains targets for compiling, building the JavaDocs, JUnit testing, and so forth. Invoke 'ant -projecthelp' for a complete list of targets.  Some of the more helpful ones include:
    • compile: compiles the code.
    • dist: creates a zip file containing a distribution of the system.
    • locc: counts the total size of the system
    • javadoc: creates the API design documentation
    • junit: runs all of the JUnit tests associated with this system.
    • checkstyle: checks the source code for comformance to various style conventions
    • clean: deletes the files created by the build process.
    • java2html: creates an HTML version of the source suitable for code reviews and browsing.
